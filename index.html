SQL Disaster Recovery

Problem: No reliable visibility into station-level production metrics.

Solution: Built a full SQL Server + SSRS reporting pipeline by parsing raw WCS log files using Bash and SQL transformations.

Impact: Delivered accurate Orders, Pieces, and Totes metrics and identified discrepancies caused by duplicate events and station restarts.

Architecture:

  • WCS Logs → Parsed via Bash (grep, awk)
  • CSV Export → Imported into SQL Server
  • SQL Views → Aggregation and normalization
  • SSRS → Final reporting layer

Data flows from raw WCS log files through a transformation pipeline into SQL Server. Bash scripts extract structured data, which is then normalized and aggregated using SQL views before being surfaced in SSRS dashboards.

Key Technique: 15-minute interval bucketing

    
DATEADD(MINUTE, DATEDIFF(MINUTE, 0, EVENT_DT)/15*15, 0)

This approach ensured accurate tracking of station activity windows instead of relying on inconsistent raw timestamps.
